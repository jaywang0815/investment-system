## 1. สร้างบัญชี advisor (เว็บ — superadmin)
- [ ] login `office.justinvestment.co` ด้วยบัญชี superadmin
- [ ] เมนู **租戶** → กล่อง **新增會員** กรอก: **公司名稱** (ชื่อบริษัทเขา) · **報告人** (ชื่อเขา) · **Email**
- [ ] กด **建立** → **ก๊อปลิงก์เชิญ** เก็บไว้
  > 公司名稱/報告人 ที่กรอก = branding บน報表ของเขา (ไม่ใช่ของ DOUU)

## 2. advisor เข้าระบบ
- [ ] ส่งลิงก์เชิญให้เขา → เขาเปิด → ตั้งรหัสผ่าน → login
- [ ] (ถ้าอยาก setup ล่วงหน้าก่อนเขาสมัคร) ใช้ปุ่ม **預先設定** ในหน้า 租戶 เข้าไปตั้งให้ได้เลย

## 3. ตั้ง LINE Bot ของเขา (ถ้าต้องการแจ้งเตือน LINE)
- [ ] **LINE Developers Console** → สร้าง **Messaging API channel** ใหม่
- [ ] **LINE OA Manager** → 設定 → 回應設定:
  - [ ] เปิด **Webhook**
  - [ ] **ปิด** 自動回應訊息 (auto-reply) ← สำคัญ! ไม่งั้นเด้ง "無法個別回覆"
  - [ ] **ปิด/แก้** 加入好友的歡迎訊息 (อย่าใช้ template ที่มี 小寶貝)
- [ ] ก๊อป **Channel secret + access token** → วางใน เว็บ **設定 → LINE 機器人** (ตอน 進入檢視 advisor คนนั้น)
- [ ] ก๊อป **Webhook URL** ที่ระบบโชว์ → ใส่ใน LINE channel + เปิด Use webhook → กด **Verify** (ต้องได้ 200)

## 4. ใส่คนรับแจ้งเตือน (LINE 管理員)
- [ ] ให้ advisor พิมพ์ `myid` หาบอตตัวเอง → ได้ LINE User ID
- [ ] เอา id ไปใส่ **設定 → LINE 管理員** (ตอน 進入檢視 เขา)
  > ⚠️ **อย่าวาง id ซ้ำ 2 รอบ** ( id ต้องยาว 33 ตัว ขึ้นต้น U) — ซ้ำแล้ว push จะ fail

## 5. Import ข้อมูลของเขา
- [ ] ขอไฟล์ Excel ของเขา
- [ ] เช็คฟอร์แมต: รองรับ **券商庫存匯出 (庫存查詢)** + DOUU block + เทมเพลตมาตรฐาน
  > ถ้าเป็นโบรกเกอร์/ฟอร์แมตใหม่ → **ส่งให้ผม (Claude) ดูก่อน** เพื่อเพิ่ม parser + ยืนยัน mapping วันที่ (โดยเฉพาะ **比價日/วันเริ่มนับดอก** = คอลัมน์ไหน — money-critical)
- [ ] **進入檢視** advisor → **匯入** → อัปโหลด → **ดู preview** (จำนวน + 比價日 ถูกไหม) → **確認匯入**
- [ ] เช็คหลัง import: จำนวนไม่ซ้ำ, 比價日 ถูก, ชื่อลูกค้าไม่มีกล่อง "?" (ถ้ามี = อักขระหายาก → แก้ชื่อในหน้า 客戶)

## 6. ตรวจสอบขั้นสุดท้าย
- [ ] advisor พิมพ์ `日報` หาบอต → เห็นข้อมูล**ของเขาเท่านั้น** (ไม่ปนคนอื่น)
- [ ] เช็ค 客戶 / 配息追蹤 / 到期管理 ว่ายอดถูก
- [ ] price alert ทำงานอัตโนมัติ (per-tenant แล้ว ไม่ต้องตั้งอะไรเพิ่ม)

## ⚠️ บทเรียน/กับดักที่เคยเจอ (กันพลาดซ้ำ)
- **โทนบอต**: advisor ใหม่ = สุภาพ (polite) อัตโนมัติ; DOUU = casual (ตั้งใน `tenants.bot_tone`)
- **เรียกชื่อลูกค้า**: ไต้หวันใช้ 姓+先生/小姐 หรือ暱稱 — ห้ามเรียกชื่อเต็ม/ห้ามใช้ 寶寶
- **比價日 ของแต่ละโบรกเกอร์ต่างกัน**: ของ 楊先生 = `最後保證配息日` (ไม่ใช่ 發行日) — ฟอร์แมตใหม่ต้องถาม advisor ยืนยัน
- **re-import หลังแก้ parser**: รอ Render deploy เสร็จก่อน + ดู preview ว่าเปลี่ยนจริง; ถ้าชื่อลูกค้าเปลี่ยน (เช่น strip อักขระ) อาจเกิด record ซ้ำ → ใช้ **合併** ในหน้า 客戶
- **แก้ DB ตรงทำไม่ได้** (classifier บล็อก) → แก้ผ่านเว็บ/re-import เท่านั้น
- **อย่าใช้ Streamlit เก่า (investment-app) เพิ่มข้อมูล** — ข้อมูลจะกำพร้า
